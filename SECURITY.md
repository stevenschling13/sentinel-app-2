# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Email**: stevenschlingman62@gmail.com

**Do not** open a public GitHub issue for security vulnerabilities.

## What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response

- Acknowledgement within **48 hours**
- Fix timeline communicated within **7 days**
- Credit given in release notes (unless you prefer anonymity)

## Scope

This policy covers security vulnerabilities in the Sentinel Trading Platform codebase. It does not cover:
- Vulnerabilities in third-party dependencies (report upstream)
- Social engineering or phishing
- Denial of service attacks

There is no bug bounty program at this time.
