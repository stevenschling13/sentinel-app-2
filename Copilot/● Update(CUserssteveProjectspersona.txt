# generateText API Reference

## Import
```typescript
import { generateText } from 'ai';
```

## Core Parameters

**model** (required)
Language model instance, e.g., `anthropic('claude-sonnet-4.5')`

**prompt** or **messages** (required)
Input prompt as string or array of conversation messages

**system** (optional)
"The system prompt to use that specifies the behavior of the model"

**tools** (optional)
"Tools that are accessible to and can be called by the model"

**toolChoice** (optional)
Options: `"auto"` | `"none"` | `"required"` | `{ "type": "tool", "toolName": string }`

**stopWhen** (optional)
"Condition for stopping the generation when there are tool results" - accepts `StopCondition` or array of conditions

## stopWhen with stepCountIs

The documentation shows `stopWhen: stepCountIs(1)` as the default behavior. This import pattern is referenced:

```typescript
import { stepCountIs } from 'ai';
```

`stepCountIs(1)` stops generation after a single step completes with tool results.

## Additional Key Parameters

- **maxOutputTokens** - token limit for generation
- **temperature/topP/topK** - sampling parameters
- **maxRetries** - retry configuration (default: 2)
- **onStepFinish** - callback when step completes
- **onFinish** - callback when entire generation completes

## Return Value

Returns object containing `text`, `toolCalls`, `toolResults`, `finishReason`, `usage`, and aggregated `totalUsage`
across all steps.

# The `tool()` Function

## Import Path
```typescript
import { tool } from 'ai';
```

## Function Signature

The `tool()` helper infers input types for the `execute` method. It accepts a configuration object with these key
parameters:

- **`description`**: string describing the tool's purpose
- **`inputSchema`**: Zod or JSON schema defining expected inputs
- **`execute`**: async function receiving typed inputs
- **`outputSchema`** (optional): schema for return type inference
- **`title`** (optional): human-readable tool name

## Complete Example

```typescript
import { tool } from 'ai';
import { z } from 'zod';

export const weatherTool = tool({
  description: 'Get the weather in a location',
  inputSchema: z.object({
    location: z.string().describe('The location to get the weather for'),
  }),
  execute: async ({ location }) => ({
    location,
    temperature: 72 + Math.floor(Math.random() * 21) - 10,
  }),
});
```

## Key Points

The documentation notes: **"location below is inferred to be a string"** — demonstrating TypeScript automatically
derives parameter types from your schema without manual type annotations.

The function returns the tool object itself, enabling seamless integration with language models for tool calling.
